# 🧠 1. مبادئ التصميم ^020801

## 1.1 المبادئ الأساسية

| المبدأ |
|-------|
| Visual-first → الرسم هو المدخل الأساسي |
| Data-driven → الواجهة تعكس البيانات |
| Non-destructive → التحذير قبل المنع |
| Context-sensitive → الواجهة تتغير حسب التحديد |
| Reusable → مكونات قابلة لإعادة الاستخدام |

---

## 1.2 فلسفة الـ Canvas

- Canvas مرئي ثنائي الأبعاد (2D)
- العناصر قد تحتوي على بيانات Logic ثلاثية الأبعاد
- العرض لا يعني المعنى التجاري

# ✅ 5. التحقق (Validation) ^020805

## 5.1 أثناء الرسم

| القاعدة |
|--------|
| منع الأشكال صفرية |
| الحد الأدنى من النقاط |
| منع Arc غير صالح |
| تحذير على تقاطع الشكل |

---

## 5.2 الخصائص

- القيم الرقمية فقط  
- الزوايا 0–360  
- الكمية ≥ 1  
- السعر ≥ 0  

---

## 5.3 Layers

- منع التكرار  
- منع حذف النشطة  
- تحذير عند إخفاء الكل  

---

## 5.4 Multi-selection

- `(mixed)` يظهر  
- التعديل يطبق على الجميع  
- إخفاء الخصائص غير المشتركة  

---

# ⚠️ 6. اعتبارات تصميمية ^020806

## Logical 3D

- فقط للعناصر الفيزيائية  
- لا يطبق على النصوص  

---

## Tags vs Marks
- نظامان منفصلان  
- يمكن توحيد البنية لاحقًا  

---

## Validation
- Engine → تحقق  
- UI → عرض الخطأ
