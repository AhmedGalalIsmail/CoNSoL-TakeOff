# 🧠 0101 — تحليل المتطلبات  
## ء Requirement Analysis

---

## 🎯 الهدف ^010100

تحديد **الرؤية التجارية، نطاق المنتج، المستخدمين المستهدفين، ومعايير النجاح** لمنصة CoNSoL وتطبيقها الرئيسي **CoNSoL‑TakeOff**.

تهدف هذه الوثيقة إلى الإجابة عن الأسئلة التالية:

| الأسئلة الرئيسية                        |
| --------------------------------------- |
| ما المشكلة التي نحاول حلّها؟            |
| ما هي منصة CoNSoL؟                      |
| ما هو تطبيق CoNSoL‑TakeOff؟             |
| من هم المستخدمون المستهدفون؟            |
| ما الذي يشمله النطاق وما الذي لا يشمله؟ |

---

## 🏗️ 1. ما هي CoNSoL؟ ^010101

**CoNSoL (Construction Solution)** هي منصة **إدارة إنشاءات معيارية** تعتمد على بنية Hub‑and‑Spoke.

### خصائص المنصة ^010101a

| المكوّن |
|--------|
| **المحرّك الأساسي — CoNSoL‑Engine** |
| - تنسيق الوحدات |
| - خدمات مشتركة |
| - إدارة التراخيص |
| - الاتصال بين الوحدات |

| الوحدة الإلزامية |
|----------------|
| **CoNSoL‑Project Manager** |
| - إدارة المشاريع |
| - الجداول الزمنية |
| - الاعتماديات |
| - تخصيص الموارد |

| الوحدات الاختيارية |
|-------------------|
| - CoNSoL‑TakeOff |
| - CoNSoL‑HR |
| - CoNSoL‑Docs |
| - وحدات مستقبلية |

### المبدأ المعماري ^010101b

> بعض الوحدات يمكن تشغيلها بشكل مستقل، بينما تعتمد وحدات أخرى على المحرّك الأساسي.

---

## 🧱 2. ما هو CoNSoL‑TakeOff؟ ^010102

**CoNSoL‑TakeOff** هو تطبيق حصر وتقدير كميات يعتمد على **الرسم كمدخل بيانات أساسي**.

### الفكرة الأساسية ^010102a

> ✏️ **الرسم ليس زخرفة — الرسم هو بيانات.**

العناصر المرسومة يتم التعامل معها على أنها:
- كائنات بيانات
- لها شكل هندسي
- لها معنى تجاري
- يمكن حساب كمياتها وتكلفتها

---

### القدرات الرئيسية ^010102b

| القدرات |
|---------|
| رسم عناصر إنشائية (جدران، بلاطات، أعمدة…) |
| ربط العناصر بالمواد |
| ربطها بالمعادلات |
| حساب الكميات تلقائيًا |
| حساب التكلفة الإجمالية |

---

### أوضاع التشغيل ^010102c

| الوضع | الوصف |
|------|------|
| Standalone | تطبيق مكتبي بقاعدة بيانات محلية |
| Integrated | مدمج داخل CoNSoL‑Engine |

---

## ❗ 3. المشكلة الحالية ^010103

### نقاط الألم الحالية ^010103a

| المشاكل |
|--------|
| الاعتماد على Excel يسبب أخطاء |
| أدوات CAD منفصلة عن التقدير |
| بطء وعدم اتساق في الحصر |
| أي تعديل يتطلب إعادة عمل |

### لماذا تفشل الأدوات الحالية ^010103b

| الأداة | القصور |
|------|--------|
| Excel | لا يوجد سياق بصري |
| CAD | لا يوجد منطق تجاري |
| برامج التقدير | لا تعتمد الرسم مباشرة |

## 👥 5. المستخدمون المستهدفون ^010105

## 🔄 6. سير العمل العام ^010106

```
إعداد → رسم → تعريف → تخزين → حساب → تقرير
```

| المرحلة | الوصف |
|-------|------|
| رسم | إدخال الشكل |
| تعريف | تحديد المعنى |
| حساب | استخراج الكميات والتكلفة |

## 🧩 7. المفاهيم الأساسية ^010107

### أوضاع الأبعاد ^0101071

| الرمز | المعنى |
|-----|-------|
| D0 | عدد |
| D1 | طول |
| D2 | مساحة |
| D3 | حجم |

### العناصر المتداخلة ^0101072
- أبواب داخل الجدران
- فتحات داخل البلاطات
- خصم المساحات تلقائيًا

---

## 📦 8. نطاق العمل ^010108

### ✅ داخل النطاق (نسخة العرض)
- لوحة الرسم ثنائية الأبعاد
- أبعاد منطقية (H×W×L)
- مواد ومعادلات
- تقدير تكلفة
- تشغيل مستقل

### ❌ خارج النطاق
- عرض ثلاثي الأبعاد حقيقي
- تعاون لحظي
- تخزين سحابي
- ذكاء اصطناعي (مستقبلاً)

---

## 📊 9. معايير النجاح ^010109

### نجاح العرض التجريبي
- رسم جدار خلال أقل من 5 دقائق
- إظهار الكمية تلقائيًا
- توليد تكلفة
- تصدير تقرير

### نجاح المنتج
- تقليل زمن الحصر
- تقليل الأخطاء
- تسريع التعديلات
